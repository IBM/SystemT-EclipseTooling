Migrating the Text Analytics properties of a project, for enabling the use of modular AQL code

If you want to use AQL files in modular format that you migrated to the current version of InfoSphere® BigInsights™ Eclipse environment, you must also migrate the Text Analytics properties of the project.

About this task

After you migrate a project that is created with InfoSphere BigInsights Version 1 (all releases) Eclipse tools to the InfoSphere BigInsights Version 2 (all releases) the projects that use Text Analytics can continue to use the AQL files in their non-modular format. However, if you want to use AQL files in modular format, you must also migrate the Text Analytics properties of the project to the newer version. After you successfully migrate the properties, you must manually restructure the code to modular AQL format.
CAUTION:
This process is irreversible. After the successful migration of the properties to modular AQL, you cannot use AQL in non-modular form, and you cannot revert the properties to a non-modular form.

Procedure

Complete one of the following tasks to migrate the Text Analytics properties of your project:
  • Right-click the project name, and select the option Migrate Text Analytics Properties.
  • In the properties menu of the project, navigate to the Text Analytics section under BigInsights, and click Migrate Properties.
