Migrating a project to the current InfoSphere BigInsights version

If you have InfoSphere® BigInsights™ projects that were created with an older version of the InfoSphere BigInsights Tools for Eclipse, you must migrate the projects to the latest version before you can use them.

About this task

Typically, projects are migrated automatically when a new version of InfoSphere BigInsights is installed. In some situations, for example, you must manually migrate a project after it is imported if it was created with an older version of the Eclipse tooling.

For projects that use Text Analytics, you want to use AQL files in modular format, you must migrate the Text Analytics properties, and then manually migrate existing AQL files. For more information, see Migrating the Text Analytics properties of a project, for enabling the use of modular AQL code.

Procedure

Perform one of the following tasks:
Option Description
Close and restart your workspace  
Manually migrate your project
  1. Right-click the project name.
  2. From the menu, select Migrate the BigInsights Project.

If you are working with multiple BigInsights servers that are at different versions, you might want your project to use a BigInsights library from a different version. In the project properties, you can change the BigInsights library version to match your server version. To publish applications or run Java™ Map Reduce programs, ensure that the BigInsights library version of the project matches the version of the selected BigInsights server.
